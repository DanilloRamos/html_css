Programação Assíncrona

  • Precisa ser utilizada quando as respostas não são obtidas de forma imediata no programa
  • Por exemplo, chamadas a uma API são assíncronas
  • Nesse caso, as execuções não ocorrem em formato de fila, e sim no seu tempo

Instrução setTimeout

  • Faz parte da programação assíncrona
  • Uma ação é estabelecida para ser executada após certo tempo
  • O código continua rodando e depois de um tempo, executa a função

Instrução setInterval

  • Semelhante à setTimeout, ou seja, é executada após um tempo
  • Porém, não para de ser executada, sua chamada é definida pelo tempo de espera na execução
  • É como um loop infinito, com execução de tempo controlada

Promises

  • São execuções assíncronas
  • Promessa de um valor que pode chegar em um ponto futuro
  • Utiliza-se o objeto promise e alguns métodos auxiliares

Tratando erros nas promises

  • Uma promise pode conter um erro, ou podemos receber um erro
  • Utiliza-se a função catch para isso, para capturar e exibir o erro

Rejeitando Promises

  • Diferente do erro, ocorre quando decide-se rejeitar uma promise
  • Utiliza-se o método reject

Resolvendo várias promises

  • Método all para executar várias promises
  • JS verifica e retorna os valores finais

Async Functions

  • Funções que retornam promises
  • Há a possibilidade de receber o resultado delas depois, além da utilização dos métodos da promise

Assertions

Instrução await

  • Serve para aguardar o resultado de uma async function
  • Torna mais simples lidar com este tipo de função, não precisando trabalhar diretamente com promise

Generators

  • Funcionam de forma semelhante às promises
  • Ações podem ser pausadas e continuadas
  • Novos operadores: function* e yield
