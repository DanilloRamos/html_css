Funções em JS

  • Estruturas menores de código
  • Ideal que tenha apenas um único objetivo
  • São reaproveitadas
  • O código pode ser dividido em várias funções
  • Uma função pode chamar outra função
  • JS já possui várias funções nativas

Estrutura da função:

  • Palavra reservada function
  • Deve possuir um nome
  • Parâmetros: ficam entre parênteses
  • Corpo da função fica entre chaves
  • Geralmente, retorna um valor

Retorno das funções

  • Retorno serve para processar um valor dentro da função e retornar para o programa
  • Palavra reservada return

Escopo das funções

  • Possuem escopo separado do restante do programa
  • Este escopo faz com que variáveis de fora não funcionem dentro
  • Novas variáveis podem ser declaradas sem interferir nas já declaradas

Escopo aninhado

  • Consiste em ter em qualquer bloco a declaração de variáveis separadas dos outros escopos
  • Um bloco é caracterizado por um código entre chaves

Arrow Function

  • Outra forma de criar funções
  • Sintaxe diferente das funções normais
  • É uma função anônima, ou seja, precisa estar em uma variável
  • Possui sintaxe mais resumida
  • As chaves e o return podem ser omitidos

Argumentos opcionais

  • Há casos de funções que podem funcionar sem algum dos argumentos
  • Checagem de parâmetro com um if

Argumentos default

  • Argumento tem valor prévio
  • Se for passado um novo valor, o default é substituído

Closure

  • Conjunto de funções, no qual temos um reaproveitamento do escopo interno de uma função
  • Podem servir para salvar os resultados já executados
  • Cria uma espécie de incrementação
  • Uma variável executa uma função e modifica o seu valor

Recursão

  • Recurso que permite a função se autoinvocar continuamente
  • É criada uma espécie de loop
  • É definida uma condição de parada, para finalizar a execução
