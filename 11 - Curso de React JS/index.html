Curso de React JS

O que é React?

  • Biblioteca JavaScript para desenvolvimento front-end
  • Categoria SPA (Single Page Application)
  • Possui a arquitetura baseada em componentes
  • Mantido pelo Facebook/Meta
  • É open source

Pré-requisitos do React

  • Precisa do Node.js
  • Gerenciador de pacotes npm para iniciar projetos

Hello World em React

  • Utilizaremos o Vite para criar aplicações
  • Digitar no terminal: npm create vite@latest e seguir as opções

Estrutura do React

    Pastas e arquivos muito importantes:

  • node_modules: dependências do projeto
  • public: assets e arquivos estáticos
  • src: código da aplicação
  • src/index.js: arquivo de inicialização da aplicação
  • src/App.js: componente principal inicial - pode ser modificado

Criando o primeiro componente

  • Se encontram em uma pasta chamada component
  • Nomeados em camel case
  • A utilização da extensão .jsx facilita a formatação pelos editores
  • Dentro do componente, é preciso criar e exportar uma função

Importando componentes

  • Para utilizar e reutilizar um component, é necessário importar
  • Sintaxe: import X from ./components/X.jsx
  • Para inserir o componente dentro de outro, utiliza-se a sintaxe de tag HTML, com o nome do componente:
  • "< FisrtComponent/ >"

JSX

  • JSX é o HTML do React
  • Há alguma diferenças do HTML, como por exemplo class = className
  • Podemos escrever JavaScript dentro do JSX
  • JSX pode ter apenas um elemento pai

Comentários no componente

  • Há duas formas de inserir comentários em React
  • Sintaxe do JS dentro e fora das funções, com //comentário
  • JSX com {/*Comentário*/}
  • Chaves são necessárias para executar qualquer instrução de JS

Template Expressions

  • Recurso que permite a execução de JS no JSX
  • Variáveis podem ser inseridas
  • Sintaxe: {a+b}
  • Tudo que vai entre chaves é entedido e executado como JS

Hierarquia de Componentes

  • Componentes podem ser reutilizados em outros
  • Uma hierarquia pode ser montada, onde um é pai do outro
  • Ao importar um componente pai, todos os outros vêm junto

Evento de click

  • React possui os mesmos eventos que JS, porém mais simples
  • Por exemplo, onClick dispara um evento que ativa uma função ao clicar em um elemento

Evento com função

  • Quando há lógicas complexas, é mais indicado criar uma função para o evento
  • Ajuda a separar as responsabilidades, facilita a manutenção

Funções de renderização

  • Funções que retornam JSX podem ser criadas
  • Podem ser utilizadas para uma renderização condicional, fazendo com que o JSX varie dependendo da operação

Aprofundando em React - Inserindo imagens

  • As imagens podem ficar na pasta public
  • Podem ser utilizadas diretamente no projeto

Importando imagens

  • Outro local comum de colocar imagens é na pasta assets
  • Nessa pasta, a imagem precisa ser importada, como um componente
  • As duas abordagens são muito utilizadas

Hooks

  • Recursos no React que possuem diversas funções
  • Podem ser criados
  • Ex.: Guardar e alterar o estado de algum dado
  • Precisam ser importados
  • Sempre começam com a palavra use
  • Alguns bem utilizados são useState e useEffect
  • Os hooks criados são chamados custom hooks
  • Toda aplicação usa pelo menos um hook

Hook useState

  • Um dos mais utilizados
  • Pode gerenciar o estados de um ou mais dados, como se fosse um getter/setter
  • Utiliza-se esse hook pois as variáveis não funcionam como esperado, não re-renderizam o componente
  • Para guardar um dado, utiliza-se setNomeDado

Renderização de lista

  • Dados do tipo array são comuns em aplicações
  • Geralmente, é recebido um array de objetos e precisa iterar nele e exibir os elementos
  • Método map fará a iteração
  • É possível inserir JSX na execução

Propriedade key

  • Iterar uma lista sem a propriedade key gera um alerta
  • React precisa de uma chave única para cada elemento
  • Isto serve para ajudar a renderização do componente
  • React utiliza para manipulação dos itens

Previous State

  • É um recurso do hook useState
  • Pode-se pegar o valor original dos dados e fazer alguma alteração
  • Muito utilizado em listas, pois modifica um ou vários itens específicos
  • O primeiro argumento do set é sempre o previous state

Renderização condicional

  • É quando parte do template é exibido por meio de uma condição
  • É um if no JSX
  • Ex de utilização: quando o usuário está autenticado ou não
  • A estrutura pode conter um else, igual à do if ternário
  • condição ? execução1 : execução2

Props

  • Recurso fundamental do React
  • Permite a passagem de dados de um componente pai para o componente filho
  • As props vêm em um objeto no argumento da função do componente

Desestruturando props

  • Componentes geralmente têm mais de uma props
  • Pode-se desestruturar no parâmetro da função do componente, para facilitar o uso
  • Sintaxe: MyComponent({name, age})

Reutilização de componentes

  • Com o auxílio da sprops, reutilizar componentes faz mais sentido
  • Torna o código padronizado, e facilita a manutenção

Reutilização de componentes com loop

  • Em arrays com muitos itens, é correto utilizar um loop para percorrer os itens

Fragments

  • São interessantes para quando há mais de um pai no componente
  • Também quando não se quer incluir HTML desnecessário no elemento pai, não alterando sua estrutura
  • Sintaxe: "<>" ""

Children Prop

  • Utilizado quando um componente precisa ter JSX dentro dele
  • JSX vem do componente pai
  • Componente age como um container, abrigando o JSX
  • A children entra como uma prop do componente

Função em prop

  • Pode-se passar funções por meio de props
  • Cria a função no elemento pai e envia como prop
  • No filho, pode-se utilizar para algum evento

State lift (Elevação de state)

  • Quando um valor é levado do componente filho ao componente pai
  • Geralmente, um componente usa o state e o outro altera
  • Componente pai gerencia os valores e passa as alterações aos filhos
