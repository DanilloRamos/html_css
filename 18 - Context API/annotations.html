O que é Context API

  • Recurso do React que facilita o compartilhamento de dados entre componentes
  • Quando há a necessidade de dados globais, usa-se o context
  • Quando há muitas idas e vindas de props, também se usa o context
  • O Context precisa ser importado onde for ser utilizado
  • Convenção para o nome do arquivo: AlgumContext.jsx

Criando o Provider

  • Provider vai delimitar o escopo do contexto
  • Provider deve encapsular os componentes que precisam do context
  • Colocado em main.jsx
  • Possui a prop children, para ser inserido dentro

Refatorando contexto com Hook

  • Um hook pode ser criado e o contexto trabalhado nele
  • O useContext é concentrado em um só local, que será no hook
  • Há um intervalo para uma possível validação na alteração do contexto

Contexto mais complexo

  • Contextos mais complexos podem necessitar de variações de comportamento
  • Para esse cenário, o mais indicado é utilizar o hook useReducer. Funciona como um useState, porém possui mais possibilidades
  • Para alterar o contexto, usa-se a função dispatch
  • Também estará no useReducer
  • É utilizado o switch
