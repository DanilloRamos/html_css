React Router

  • Pacote mais utilizado para criar rotas em uma aplicação React
  • Cada rota é uma página
  • Precisa ser instalado e configurado no projeto
  • Possui diversas funcionalidades, como redirecionamento, rotas aninhadas, 404, etc

Configurando o React Router

  • Utiliza-se principalmente o arquivo main.jsx
  • Os componentes createBrowserRouter, RouterProvider e Route são importados
  • Serão utilizados na configuração e ao longo do projeto

Configurando página de erro 404

  • Componente ErrorPage
  • Hook useRouteError para obter as informações de erro
  • Propriedade errorElement é configurada no main.jsx

Criando componente base

  • Outlet é um componente que permite reaproveitar a estrutura das páginas
  • Um cmponente base pode ser sua estrutura, e todas as páginas ficam dentro dele
  • As configurações são feitas na propriedade children, dentro de main.jsx

Links entre páginas

  • Utiliza-se o componente link
  • Configurado com a propriedade to, que leva à URL de destino. Isso permite uma mudança de páginas sem reload

Carregando dados

  • O hook useFetch é utilizado para exibir dados na home

Rotas dinâmicas

  • Recurso de carregar rotas individuais
  • A URL de cada item irá variar, de acordo com alguma característica
  • Formato do path: /product/:id

Rotas aninhadas

  • Nested routes, são estruturas mais complexas
  • Combinam-se as rotas dinâmicas para serem criadas estruturas maiores de acesso
  • Ex.: /products/:id/info

Link ativo

  • Componente de NavLink ao invés de Link, para identificar links ativos
  • Propriedade isActive para aplicar estilos diferenciados para este link

Search Params

  • Esse recurso permite pegar informações da URL
  • Interessante para funcionalidade de busca em site
  • useSearchParams, hook para resgatar dados

Redirect

  • Redirecionamento de páginas é criado com o componente Navigate
  • Configuração é feita no main.jsx
