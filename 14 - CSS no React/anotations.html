CSS Global

  • Utilizado para aplicar estilos a todos os elementos do projeto
  • Arquivo index.css, na pasta src

CSS de Componente

  • Utilizado apenas em componentes específicos
  • Pode vazar para outros elementos do projeto
  • Geralmente o arquivo é criado com o mesmo nome do componente

Inline Style

  • Igual ao do CSS
  • Por meio do atributo style, aplicam-se regras de CSS diretamente a um elemento

CSS inline dinâmico

  • Técnica de aplicação de estilo por condição
  • Atributo é inserido em um if ternário
  • Dependendo da condição e do seu resultado, um estilo diferente pode ser exibido

Classes dinâmicas

  • Uma lógica de aplicar classes a elementos pode ser aplicada
  • If ternário
  • Mais interessante que o CSS inline, pois o conteúdo da classe estará no arquivo CSS

CSS Modules

  • Permite deixar o CSS num escopo
  • Nome do arquivi: Component.module.css
  • Necessário fazer importação

Formulários no React

  • Em React, é utilizada a tag form
  • As labels tem o nome alterado para htmlFor, que conta com o nome do input
  • Não é utilizado o action, o envio deve ser feito pelo JS de forma assíncrona

Label envolvendo input

  • Um padrão muito utilizado é a label ser pai do input
  • Atributo for é opcional
  • Simplifica o HTML e mantém a semântica

Manipulando valores

  • Para manipular valores de formulários, utiliza-se o hook useState
  • O valor é armazenado com o set
  • O evento que vai nos inputs é o onChange, no qual tem a função de alteração
  • Quando há diversos inputs, cria-se uma função inline dentro do onChange

Envio de formulário

  • Evento onSubmit
  • Funções podem ser executadas no evento
  • O envio do formulário tem que ser parado com preventDefault
  • As validações são feitas nesta função

Controlled Inputs

  • Valores pré-existentes podem ser atribuídos aos formulários
  • Pode-se igualar ao atributo value do state
  • Se não houver valor, pode haver uma lógica que entrega uma string vazia

Limpando formulários

  • Basta atribuir valores vazios aos states

Textarea

  • Pode ser aplicado como um input normal de texto
  • Atributo value pode ser utilizado para mudar o texto inicial
  • Com o onChange, muda-se o state

Input Select

  • Muito semelhante aos outros inputs
  • onChange pode mudar o valor do state
  • Value deve ser atribuído a uma das options
